# Advanced Tailwind CSS Tips and Tricks

Tailwind CSS is a powerful utility-first framework. Here are some advanced techniques to make the most of it.

## Custom Component Patterns

### Button Component
```jsx
const Button = ({ variant = 'primary', size = 'md', children }) => {
  const variants = {
    primary: 'bg-blue-500 hover:bg-blue-600 text-white',
    secondary: 'bg-gray-500 hover:bg-gray-600 text-white',
    outline: 'border-2 border-blue-500 text-blue-500 hover:bg-blue-50'
  };

  const sizes = {
    sm: 'px-3 py-1.5 text-sm',
    md: 'px-4 py-2 text-base',
    lg: 'px-6 py-3 text-lg'
  };

  return (
    <button className={`
      ${variants[variant]}
      ${sizes[size]}
      rounded-lg transition-colors duration-200
      focus:ring-2 focus:ring-offset-2 focus:ring-blue-500
      disabled:opacity-50 disabled:cursor-not-allowed
    `}>
      {children}
    </button>
  );
};
```


## Performance Tips

1. Use JIT mode for faster development
2. Purge unused styles in production
3. Extract common patterns into components
4. Use @apply for complex, reusable styles
5. Leverage Tailwind's built-in variants

## Best Practices

### Using @apply Wisely
```css
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-500 text-white rounded-lg
    hover:bg-blue-600 focus:ring-2 focus:ring-blue-500
    focus:ring-offset-2 transition-colors duration-200;
  }
}
```

### Organizing Classes
```html
<button class="
  /* Layout */
  inline-flex items-center justify-center
  /* Typography */
  font-semibold text-sm
  /* Spacing */
  px-4 py-2
  /* Colors */
  bg-blue-500 text-white
  /* Effects */
  hover:bg-blue-600
  focus:ring-2 focus:ring-blue-500 focus:ring-offset-2
  /* Transitions */
  transition-all duration-200
">
  Click me
</button>
```

Remember: Tailwind is most powerful when you embrace its utility-first nature while maintaining clean, organized code through smart component abstractions.
